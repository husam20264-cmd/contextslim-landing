ContextSlim is a local context engineering CLI that reduces token waste and builds task-specific context packs for AI coding tools.

The Problem

AI coding tools often receive too much noisy context: irrelevant files, long chat history, duplicated code, and missing task-specific snippets.

The Solution

ContextSlim scans your repo and builds a focused context pack for the current task.

contextslim build ./repo --task "fix login bug"

It generates:

.contextslim/project_map.json

.contextslim/final_context.md
.contextslim/token_report.md

Features

Core Features

FeatureDescription
Repo IndexingScans your entire project, extracts file metadata, purpose, and important functions
Task-Based SelectionUnderstands your task and selects only relevant files
Smart Relevance ScoringScores files by: path (+10), purpose (+5), summary (+2), functions (+2 each), function name match (+12)
Real Code SnippetsExtracts actual lines of code around task keywords (3 lines before, 3 after)
Keyword ExpansionExpands task keywords with synonyms (fix → repair, resolve, correct)
Smart CompressionCompresses large files while preserving important functions (target: 800 tokens each)
Dynamic Overhead5% of total tokens (min 50, max 300) — not a fixed +2000
Token Budget ReportShows before/after tokens, savings percentage, and cost impact
Organized OutputAll outputs in .contextslim/ folder: project_map.json, final_context.md, token_report.md

Technical Features

FeatureDescription

Function-Aware ScoringDetects functions and classes, uses them for relevance scoring
Agent IsolationSupports multiple agent types: code, docs, security, devops
Snippet FallbackUses function names as fallback when no snippet is found
Termux SupportWorks on Android devices via Termux
